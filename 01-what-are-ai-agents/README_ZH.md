# 第一课：什么是 AI 智能体？

## 简介

你可能用过 ChatGPT、Gemini 或 Claude 来回答问题或写代码。你输入内容，得到回复，然后继续。这是语言模型在发挥作用——根据你的输入预测有用的文本。

AI 智能体则不同。智能体能够**思考**、**行动**和**记忆**。它不只是回答你的问题，而是自行规划所需步骤，借助工具完成这些步骤，并根据过程中的反馈不断调整策略。

换个角度想：如果你雇了一位工程师，却只允许他说话，不让他动键盘、打开浏览器或查阅文档，他能做的事情会非常有限。这就是单独运行的 LLM。现在给这位工程师访问你的代码库、终端、公司文档以及提问的权限——这就是智能体。

本课将介绍什么是 AI 智能体、它与普通语言模型的区别、使其运转的组件，以及何时应该（或不应该）使用智能体。

---

## 用大白话解释：什么是 AI 智能体？

AI 智能体是一种以语言模型为核心推理引擎、并具备在现实世界采取行动能力的软件系统。这些行动可能包括：

- 搜索网络
- 查询数据库
- 调用 API
- 读写文件
- 发送邮件
- 运行代码

关键区别在于**自主性**。普通 LLM 响应单条提示。智能体接收一个目标，然后自行决定采取哪些步骤、执行这些步骤、观察结果，并持续运行直到目标达成（或判定目标无法达成）。

### 新员工类比

想象你雇了一位新工程师。第一天，你不会期望他什么都知道，但你会期望他：

1. **阅读文档**，了解代码库
2. **使用工具**，如 IDE、终端和浏览器
3. **提出问题**，遇到不明白的地方
4. **拆解任务**，把大任务分解成小步骤
5. **检查工作成果**，确认完成后再汇报
6. **从错误中学习**，调整方法

AI 智能体的工作方式相同。它拥有知识基础（语言模型）、工具访问权限，以及管理思考-行动-观察循环的编排层。

## LLM 与智能体：有何区别？

这是最重要的区别，需要尽早内化。

| 方面 | 单独的 LLM | AI 智能体 |
|---|---|---|
| **功能** | 根据提示生成文本 | 通过多步骤追求目标 |
| **交互方式** | 单轮（或多轮对话） | 自主循环的思考与行动 |
| **工具** | 无——文本输入，文本输出 | 可调用函数、API、搜索等 |
| **记忆** | 仅限于上下文窗口 | 可在步骤间持久化信息 |
| **决策** | 响应你的提问 | 自行决定下一步 |
| **错误处理** | 给出答案（对错不论） | 可观察错误并以新方法重试 |

一个有用的思维模型：

- **LLM** = 大脑
- **智能体** = 大脑 + 双手 + 记忆

大脑（LLM）负责推理，双手（工具）使其能够行动，记忆（状态管理）使其能追踪已发生的事情和待完成的事项。

### 具体示例

**单独 LLM：** 你问"GOOG 股票现在的价格是多少？" 模型可能会说"根据我最新的训练数据，大约是 140 美元"——这可能已经是几个月前的数据了。

**智能体：** 你问同样的问题。智能体思考"我需要获取当前股票数据，应该用金融 API。"它调用股票价格工具，获取实时价格，返回准确答案。如果 API 调用失败，它可能尝试其他数据源。

这个循环——思考、行动、观察、重复——正是智能体区别于普通模型的本质。

## AI 智能体的核心组件

无论采用何种框架，每个智能体系统都有三个基本组件：

### 1. 模型（大脑）

这是智能体核心的语言模型，负责：

- **理解**用户目标
- **推理**需要采取的步骤
- **决定**调用哪个工具（以及用什么参数）
- **解读**工具调用的结果
- **生成**最终回复

模型的选择至关重要。复杂任务（多步骤推理、复杂代码生成、细致的决策判断）受益于 Gemini 或 Opus 等前沿模型。简单任务（分类、提取、直接问答）可以使用 Gemini Flash 等轻量模型以节省成本和降低延迟。

### 2. 工具（双手）

工具使智能体能够与文本生成之外的世界进行交互。没有工具，智能体只是个聊天机器人。有了工具，它就能：

- **检索信息**：搜索网络、查询数据库、读取文件
- **执行操作**：发送邮件、创建工单、部署代码
- **计算处理**：运行计算、执行代码、转换数据

工具通常定义为具有明确名称、描述和参数模式的函数。模型决定何时以及如何调用它们。我们将在第三课深入介绍工具。

### 3. 编排层（控制循环）

这是连接模型和工具、使系统正常运转的"胶水"。编排层负责管理：

- **智能体循环**：思考 -> 行动 -> 观察 -> 重复
- **状态管理**：已发生的事情，模型需要的上下文
- **错误处理**：工具调用失败时的应对措施
- **终止条件**：何时停止循环并返回结果
- **护栏**：安全检查、输出验证、范围限制

最简单的编排模式如下：

```
1. 接收用户目标
2. 将目标 + 可用工具发送给模型
3. 模型返回：
   a. 最终答案 -> 返回给用户
   b. 工具调用 -> 执行工具，将结果添加到上下文，返回步骤 2
```

这通常称为 **ReAct 循环**（推理 + 行动）。更复杂的模式将在后续课程中探讨。

### 组件如何协同工作

```
用户目标
    |
    v
+-------------------+
| 编排层            |
|                   |
|  +-------------+  |
|  |   模型      |  |    "我需要搜索 X"
|  |  （大脑）   |--+--->  工具调用
|  +-------------+  |         |
|        ^          |         v
|        |          |  +-------------+
|        +----------+--+   工具      |
|     工具结果      |  |  （双手）   |
|                   |  +-------------+
+-------------------+
    |
    v
最终回复
```

## 智能体系统分类

并非所有智能体都一样。从第 0 级到第 4 级，可以按自主性和能力的高低来理解智能体系统。

### 第 0 级：基础推理（简单 LLM）

**特点：** 语言模型回答问题，无工具或记忆。

**示例：** 你问 Gemini"解释 CAP 定理"，它从训练数据中给出清晰解释。

**能力：**
- 文本生成与理解
- 单轮或多轮对话
- 无外部数据访问
- 无法采取行动

**适用场景：** 通用知识问题、创意写作、头脑风暴、已提供文本的摘要。

### 第 1 级：互联问题解决者（工具使用型智能体）

**特点：** 可调用工具检索信息或执行简单操作。这是从"聊天机器人"跨越到"智能体"的界限。

**示例：** 可通过调用订单 API 查询订单状态的客服机器人，或能搜索文档的编程助手。

**能力：**
- 第 0 级的全部能力
- 函数调用（工具）
- 检索增强生成（RAG），以真实数据为基础
- 简单的单步或少步任务完成

**适用场景：** 需要当前数据、API 集成、步骤较少的简单工作流程。

### 第 2 级：战略型智能体（具备上下文的自主体）

**特点：** 可以规划多步骤方法，在较长会话中保持上下文，并根据中间结果调整策略。

**示例：** 一个研究智能体接到"比较三大云服务商的无服务器定价"的问题，然后搜索定价页面、提取数据、构建对比表格，并总结发现。

**能力：**
- 第 1 级的全部能力
- 多步骤规划与执行
- 情况变化时的动态重规划
- 跨步骤的工作记忆
- 自我评估（"这个结果够好吗？"）

**适用场景：** 研究任务、复杂排障、路径依赖中间结果的多步工作流程。

### 第 3 级：协作多智能体系统

**特点：** 多个专业智能体协同工作，各自负责更大任务的不同方面，由一个智能体负责协调。

**示例：** 一个软件开发系统，其中一个智能体写代码，另一个写测试，第三个审查代码，编排智能体管理整个工作流程。

**能力：**
- 第 2 级的全部能力
- 智能体间通信
- 专属角色与任务委派
- 子任务并行执行
- 质量共识或投票机制

**适用场景：** 受益于专业化的复杂项目，需要多视角或质量门控的任务。

### 第 4 级：自我进化智能体

**特点：** 可以反思自身表现、从历史运行中学习、更新策略，并在无人工干预的情况下持续改进。

**示例：** 一个部署智能体追踪历史上哪些回滚策略效果最好，并据此调整未来的部署方式。

**能力：**
- 第 3 级的全部能力
- 长期记忆与学习
- 基于历史结果的策略优化
- 提示词或工具选择的自我修改
- 性能监控与自我修正

**适用场景：** 随时间出现规律的重复性任务，受益于持续改进的系统。

### 汇总表

| 级别 | 名称 | 核心特性 | 示例 |
|---|---|---|---|
| 0 | 基础推理 | 文本输入输出 | 聊天机器人、问答 |
| 1 | 互联问题解决者 | 工具使用 | 订单查询机器人 |
| 2 | 战略型智能体 | 多步骤规划 | 研究助手 |
| 3 | 协作多智能体 | 智能体协调 | 开发团队模拟 |
| 4 | 自我进化 | 从经验中学习 | 自适应运维智能体 |

今天大多数生产级智能体系统运行在第 1 级或第 2 级。第 3 和第 4 级正在成为实践中日益可行的选择，但会增加显著的复杂性。从简单开始，只有当你有明确理由时才向上升级。

## 何时使用智能体，何时一个简单的提示就够了

智能体带来强大能力，但也带来复杂性、成本和延迟。并非每个问题都需要智能体。以下是实用指南。

### 使用简单提示的场景：

- 任务可以在单步内完成
- 不需要外部数据或操作
- 答案存在于模型的训练数据中
- 对低延迟有严格要求（智能体会增加多次往返）
- 多次模型调用的成本不合理

**示例：**
- "总结这段文字"
- "把这段 JSON 转换成 Python 数据类"
- "写一个匹配邮件地址的正则表达式"
- "解释 TCP 和 UDP 的区别"

### 使用智能体的场景：

- 任务需要相互依赖的多个步骤
- 需要外部数据或工具（API、数据库、搜索）
- 任务需要实时或最新信息
- 根据中间结果可能需要调整方法
- 任务涉及采取行动（不仅仅是生成文本）

**示例：**
- "查找我们问题追踪系统中最近三个 Bug，并为团队站会起草摘要"
- "查看客户的订单，检查配送状态，并发送更新邮件"
- "调研竞争对手定价并建立对比表格"
- "审查这个拉取请求，运行测试，并提出改进建议"

### 决策流程图

```
任务需要外部数据或操作吗？
  |
  +-- 否 --> 模型能从训练数据中回答吗？
  |            |
  |            +-- 是 --> 使用简单提示
  |            +-- 否 --> 先考虑 RAG（检索），然后考虑智能体
  |
  +-- 是 --> 只是单次工具调用吗？
               |
               +-- 是 --> 简单的函数调用设置可能就够了
               +-- 否 --> 使用带编排的智能体
```

### 成本与延迟的考量

智能体循环中的每个步骤都涉及一次模型调用。一个 5 步智能体工作流意味着 5 次以上对模型的调用，加上工具执行时间，这会叠加：

- **延迟**：每次模型调用需要 1-10 秒，取决于模型和提示大小。一个 5 步智能体可能需要 15-30 秒。
- **成本**：每次模型调用都消耗 token。智能体工作流的 token 用量可能是单次提示的 10-50 倍。
- **可靠性**：步骤越多，出错或产生幻觉的机会越多。

工程原则与其他地方相同：使用能完成工作的最简单方案。

## 真实世界案例

### 客服智能体

**目标：** 端到端处理客户咨询。

**工作流程：**
1. 客户写道："我的订单 #12345 在哪里？"
2. 智能体使用订单 ID 调用订单查询工具
3. 获取状态："已发货，追踪号 XYZ，预计 3 月 20 日送达"
4. 智能体格式化包含追踪链接的友好回复
5. 如果客户要求更改收货地址，智能体调用地址更新工具

**级别：** 1-2（工具使用加少量多步骤逻辑）

### 代码助手智能体

**目标：** 帮助开发者编写、调试和改进代码。

**工作流程：**
1. 开发者问："为什么这个函数返回 null？"
2. 智能体读取相关源文件
3. 搜索相关测试
4. 发现 Bug（第 42 行缺少 null 检查）
5. 提供带代码的修复建议
6. 可选地运行测试以验证修复有效

**级别：** 2（带工具使用的多步骤推理）

### 研究智能体

**目标：** 从多个来源收集和综合信息。

**工作流程：**
1. 用户问："2026 年服务端渲染的优缺点是什么？"
2. 智能体搜索最近的文章和基准测试
3. 阅读并提取多个来源的关键点
4. 交叉引用观点并检查一致性
5. 生成带引用的结构化摘要

**级别：** 2（跨多步骤搜索、阅读、综合）

### DevOps 事件响应智能体

**目标：** 帮助诊断和解决生产事故。

**工作流程：**
1. 告警触发："auth-service 的 API 延迟激增"
2. 智能体查询过去 30 分钟的监控仪表板
3. 检查最近的部署变更
4. 检查日志中的错误模式
5. 关联发现："延迟激增在 #789 部署（更改了 auth token 缓存 TTL）后 5 分钟开始"
6. 建议回滚并起草事故报告

**级别：** 2-3（多步骤调查，可能与其他智能体协调）

## ELI5：什么是 AI 智能体？

### 把智能体想象成一个超级能干的实习生

想象一下你有一个刚入职第一天的实习生。他很聪明，毕业时名列前茅，但从没见过你的代码库。

**单独的 LLM 就像这位实习生坐在没有电脑的房间里。** 你可以问他问题，他会根据在学校学到的知识给出有见地的答案。但他无法查资料、运行代码或发送邮件。他能做的只是说话。

**智能体就像这位实习生有了完整的工作台配置。** 他有笔记本电脑、访问内部工具的权限、浏览器和公司 Slack。现在当你问他问题时，他可以：

- 不知道答案时查资料
- 尝试运行代码来验证想法
- 查阅文档确认自己的判断
- 向同事（另一个智能体）寻求帮助
- 带着经过验证的答案回来告诉你

这位实习生有时也会犯错——毕竟他是新人。但他能发现大部分错误，因为他能检查自己的工作。遇到困难时，他知道该求助，而不是凭空猜测。

**核心洞察：** 实习生的大脑没有改变。改变的是他能访问什么以及他如何开展工作。这正是 LLM 和智能体的区别——相同的大脑，更多的能力，更好的流程。

## 核心要点

1. **AI 智能体是一个使用语言模型推理、借助工具行动、通过编排层管理思考与执行之间循环的系统。**

2. **LLM = 大脑。智能体 = 大脑 + 双手 + 记忆。** 模型提供推理能力，工具提供行动能力，编排层提供控制流程。

3. **智能体存在于一个谱系中**，从简单的工具使用型助手（第 1 级）到自我进化系统（第 4 级）。从能解决问题的最低级别开始。

4. **并非所有事情都需要智能体。** 如果单次提示就能完成工作，就用单次提示。只有当任务确实需要工具、多步骤推理或现实世界操作时，才添加智能体能力。

5. **核心循环很简单：** 接收目标 -> 思考下一步 -> 使用工具 -> 观察结果 -> 重复直到完成。
